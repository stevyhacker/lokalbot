# OCR Benchmark Results - 2026-06-24

Input set: real LokalBot stored screenshots exported from the encrypted local
store to `/private/tmp`, using `ExportVisionBenchmark.swift`. Text content was
not copied into this file.

Hardware/runtime: Apple M4 Max MacBook, local CPU/MPS/Metal where available.

| Engine | Images | Mean latency | Notes |
| --- | ---: | ---: | --- |

## Runtime Notes

- DeepSeek-OCR official Transformers code is CUDA-oriented and hard-codes
  `.cuda()` / `torch.autocast("cuda")`; the macOS-local path tested here was
  `ggml-org/DeepSeek-OCR-GGUF`.
- GLM-OCR's README requires installing Transformers from GitHub; stable
  Transformers could not instantiate the processor for the checked repo.
- PaddleOCR-VL and DeepSeek GGUF were run through the bundled `Vendor/llama-cpp/llama-server`
  with multimodal projector support.

## Recommendation

Keep Apple Vision as the default OCR pipeline. None of the tested open-source
models improved both speed and quality on real LokalBot screenshots. The only
candidate worth keeping around for future experimentation is PaddleOCR-VL GGUF,
but as a secondary/manual "document parse" mode, not periodic screenshot OCR.
